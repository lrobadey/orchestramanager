# Orchestra Manager Design Principles

## Institutional Inertia

Institutions accumulate history.

Repertoire familiarity, audience expectations, donor relationships, section culture, artistic reputation, library ownership, musician pipelines, financial scars, and programming traditions should create long-term momentum that cannot be instantly reversed.

The player influences the institution, but does not fully command it.

The orchestra should feel like a large system with memory and inertia rather than a perfectly reactive strategy sandbox.

## The Orchestra as a Living Organism

The orchestra should feel like a living institution with evolving strengths, weaknesses, traditions, habits, stylistic reflexes, and biases rather than a neutral collection of interchangeable modifiers.

Section cultures should emerge over time.

Programming history should matter.

Audience expectations should drift.

Musician pipelines and institutional identity should shape future possibilities.

The institution should develop a recognizable character through repeated decisions.

## AI as Interpretation, Not Authority

Deterministic simulation systems remain authoritative over canonical game state.

LLM systems should interpret, narrativize, critique, negotiate, contextualize, and communicate simulation outcomes rather than directly owning simulation truth.

AI-generated critics, donors, composers, musicians, administrators, and reviewers should respond to validated simulation events.

The game should avoid replacing systemic causality with unconstrained improvisation.

## Identity as a First-Class System

Institutional identity is not cosmetic flavor.

Identity should materially alter future strategic affordances:

- which musicians are attracted to the orchestra
- which audiences trust it
- which donors support it
- which collaborations become viable
- which commissions feel coherent
- which programming risks feel credible
- how critics interpret decisions

The orchestra should slowly become known for something.

## Culture as a Simulated System

Orchestra Manager is not only an operational management simulation.

It is also a simulation of cultural positioning, artistic legitimacy, institutional reputation formation, and audience trust inside a broader artistic ecosystem.

The game should model not just finances, but how cultural institutions accumulate meaning, prestige, expectation, and ideological identity over time.
