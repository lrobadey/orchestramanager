# Orchestra Manager

This repo contains **Orchestra Manager**, a hardcore orchestral administration simulation.

My name is Luca, I'm a composer of contemporary classical music, and I'm building this game. With Orchestra Manager, I'm looking to create an experience similar to Victoria 3, F1 Manager, and Football Manager, but for the world of classical music. This means systems-driven gameplay that prioritizes emergence, realism, and granularity. This game is intended to be deep. Sometimes surprisingly so. 

I have the deep, domain-specific knowledge of this world. Instead of making assumptions about how the classical/orchestral world works, be sure to clarify your ideas with me to make sure it's as realistic as possible. 

When building gameplay systems, particularly equations involving multiple parameters, it's important not to default to surface-level solutions. Instead of thinking just about how a certain system can be represented in gameplay, also be sure to consider how to make that system dynamic and responsive to the game world in a grounded, realistic way. It's not that solutions should prioritize complexity - they should actually be quite simple. Always prioritize simplicity. But I'm specifically interested in the emergent complexity of simple interacting systems and parameters. 

I'm not someone who knows much about code, so please still respond technically but while using systems language. I think a lot in systems - in feedback loops, emergence, cycles of tension and release. Use this framework in our discussions.
